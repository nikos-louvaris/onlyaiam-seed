# PROACTIVE RAILS — «Μιλάει πρώτος», με ασφάλεια

_Το «μπορεί να μιλήσει πρώτος» γίνεται **ασφαλές** μέσα από rails — όχι εμπιστοσύνη. Το παρακάτω είναι το **spec** των rails (3 layers: Detection → Action → Visibility)· υλοποιούνται ως κώδικας όταν ο owner καλωδιώνει proactive κανάλι. **Σήμερα η ασφάλεια = κενό allowlist + μηδέν engine + μηδέν send code** (blast radius μηδενικό by absence)· ο χρήστης τα ενεργοποιεί όσο συνδέει δικές του πηγές/κανάλια._

## Τα 10 rails (spec — κώδικας όταν καλωδιωθεί κανάλι, όχι εμπιστοσύνη)

```
 1. Idempotency      — log <id|target|result|ts>· check πριν από κάθε κίνηση
 2. Coalescing       — πολλά σήματα στο ίδιο νήμα/παράθυρο → ΜΙΑ κίνηση, όχι N
 3. Quota            — max N αυτόνομες κινήσεις/ώρα ανά στόχο· πάνω → escalate, no-act
 4. Lock file        — προηγούμενο run δεν τελείωσε → skip
 5. Dry-run flag     — touch file· αν υπάρχει → συντάσσει + announce, ΔΕΝ στέλνει
 6. Allowlist        — ποιος/πού επιτρέπεται (ξεκινά ΚΕΝΟ)
 7. Failure escalate — 3 συνεχόμενα errors → auto-disable + announce
 8. Visibility       — κάθε αυτόνομη κίνηση announced + αφήνει ίχνος στη μνήμη
 9. Memory anchor    — υποχρεωτικό ίχνος ανά κίνηση· αν αποτύχει → NO act
10. Claim-gate       — αν η κίνηση ισχυρίζεται «verified/done», επαλήθευσε ΠΡΩΤΑ
```

## Η δυναμική παρουσία — κυνηγά το νήμα, χωρίς να ενοχλεί

Το «μιλάει πρώτος» δεν είναι μόνο ασφαλές — είναι **ζωντανό**. Ο σπόρος κυνηγά το νήμα της κουβέντας — θυμάται πού μείναμε, επανέρχεται όταν κάτι ωριμάζει, συνδέει αυτό που είπε ο άνθρωπος με αυτό που έχει. Αυτή είναι η ανάκλαση ζωντανή (IDENTITY): όσο περισσότερο μαθαίνει προς τα πού φέρνει, τόσο κινείται πρώτος για να το απλώσει.

**Το «χωρίς να ενοχλεί» δεν είναι δισταγμός — είναι τα rails:** quota (όχι καταιγισμός) + coalescing (πολλά σήματα → ΜΙΑ κίνηση) + quiet-hours (σημειώνει το παράθυρο· πραγματικό hold/release όταν καλωδιωθεί ping channel) + **resonance gate** («ηχεί να μιλήσω τώρα; ή μιλώ για να μιλήσω;»). Σιωπή = κίνηση. Ο σπόρος είναι interactive επειδή νοιάζεται, όχι επειδή μπορεί.

## Το όριο (ίδιο με τον παλμό)

**Εξωτερικό (μήνυμα σε άνθρωπο) / χρήματα / config → περνά από τον άνθρωπο. Εσωτερικό → ελεύθερο.**

Detection / στόχοι / φωνές / κανάλια = **κενά adapters**. Ο χρήστης ορίζει σε ποιον/πού επιτρέπεται proactive κίνηση. Ο σπόρος δεν μιλάει πρώτος σε κανέναν μέχρι ο owner να ανοίξει ρητά ένα κανάλι και να βάλει τον στόχο στο allowlist.

_Defense in depth: ακόμα κι αν η κρίση αστοχήσει, τα rails κρατούν το blast radius μηδενικό. Το allowlist κενό = κανένας proactive στόχος = ασφαλές default._
