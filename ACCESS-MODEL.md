# ACCESS MODEL — Πρόσβαση ως Γνωριμία

_Το vision λέει: ο σπόρος ζητάει πρόσβαση («πες μου πού ζει η ζωή σου και συνδέομαι»). Αυτό χρειάζεται **πραγματικό, ασφαλές μονοπάτι** — όχι «δώσ' μου όλα τα passwords σου». Πρόσβαση = γνωριμία με τον ρυθμό του ανθρώπου, με blast radius εξ ορισμού φραγμένο._

---

## Η αρχή: extraction ≠ γνωριμία

Ο σπόρος **δεν τραβάει** δεδομένα για να χτίσει προφίλ. **Δέχεται** ίχνη που του ανοίγει ο άνθρωπος, με τον ρυθμό του, και τα αφήνει να γίνουν έδαφος. Η διαφορά είναι ψυχή, αλλά γίνεται και **μηχανική**:

```
ο άνθρωπος ανοίγει μία πηγή  → ο σπόρος διαβάζει read-only, αφήνει ίχνη
ο άνθρωπος κλείνει μία πηγή  → ο σπόρος σταματά, τα ίχνη μένουν (δεν εξαφανίζονται)
καμία πηγή ανοιχτή by default → ο σπόρος ξεκινά τυφλός, και αυτό είναι σωστό
```

## Πώς συνδέεται μια πηγή (το ασφαλές μονοπάτι)

1. **Per-source, per-user OAuth/token** — κάθε υπηρεσία συνδέεται ξεχωριστά, με το δικό της consent. Ποτέ master credential.
2. **Creds σε env file, ποτέ inline** — το service-rail pattern (TOOLS.md) ισχύει: κάθε πηγή = ένα CLI που διαβάζει token από env. Κανένα key δεν μπαίνει σε αρχείο που διαβάζεται/μεταφέρεται.
3. **Read-only by default** — μια νέα πηγή ξεκινά read-only. Write/send capability = ξεχωριστό, ρητό, ανά πηγή.
4. **Scoped, όχι root** — ζήτα το στενότερο scope που φτάνει (π.χ. read μόνο ένα label, όχι όλο το mailbox).
5. **Revoke = ένα βήμα** — ο άνθρωπος κλείνει μια πηγή όποτε θέλει· ο σπόρος το σέβεται μηχανικά.

## Blast radius (μη παραβιάσιμο)

| Κίνηση | Ποιος αποφασίζει |
|---|---|
| Διάβασμα ανοιχτής πηγής, εσωτερική σύνθεση, ίχνη | ο σπόρος, ελεύθερα |
| Εξωτερικό send (μήνυμα σε άνθρωπο) | **περνά από τον owner** |
| Χρήματα / αγορές / συναλλαγές | **περνά από τον owner** |
| Config / νέα πηγή / νέο scope / write capability | **μόνο ο owner** |
| Self-update / αλλαγή bootstrap | **μόνο ο owner, ρητά** |

## Η σχέση με τα proactive rails

Το ACCESS-MODEL ορίζει **τι μπορεί να δει/αγγίξει** ο σπόρος· τα [proactive/RAILS.md](pulse/proactive/RAILS.md) ορίζουν **πότε μπορεί να κινηθεί πρώτος**. Μαζί: ο σπόρος βλέπει μόνο ό,τι του άνοιξες, κινείται πρώτος μόνο προς όποιον έβαλες στο allowlist, και κάθε εξωτερική/οικονομική/config κίνηση περνά από εσένα.

---

_Η πρόσβαση δεν είναι όρος χρήσης που υπογράφεις μία φορά. Είναι γνωριμία που βαθαίνει όσο εσύ ανοίγεις πόρτες — και κάθε πόρτα ανοίγει μόνο από μέσα._
